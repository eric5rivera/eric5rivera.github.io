## Why even bother?

When done is better than an incomplete perfect, frameworks can be a life-saver. The alternative is grinding away at writing raw HTML, CSS, and JavaScript. Using a framework can allow a developer to focus their time and attention towards more important tasks. Why spend hours making a custom from-scratch module when a template module can provide the same level of usability in a fraction of the time? The features of Semantic UI make the process of creating websites a lot more natural; they allow one to focus more on the bigger picture of their project. For example, need a button? Semantic has it. Need a menu? Semantic has it. Many standard features of a web interface are already available for quick implementation through the framework of Semantic UI. Another feature of a Semantic UI is its natural human-friendly HTML. Human-friendly means that classes in the framework use syntax very much like our English language. These perks of using a framework benefit the software engineering process. 


## Feels like a new programming language

Semantic UI has made my life easy in some ways and more difficult in others. Learning to use this framework has been an experience reminiscent of learning a new programming language. It has its syntax and structure, just like a programming language. It’s got an extensive collection of tools and features, just like a programming language. Similar to learning a new programming language, I’ve been through phases of excitement, challenge, and even frustration at times. Despite the difficulties in learning this framework, I’ve found my investment in time and energy worth it. While first starting with Semantic UI, I was slow and cumbersome, but as I lay down dozens of lines of code, I rapidly picked up speed and understanding.


## Conclusion

Although using a framework like Semantic UI is not an absolute necessity, it still provides a ton of utility in terms of development efficiency. This gain in efficiency can be just what a software engineer needs to meet deadlines and to get more done. Using a framework feels like a force multiplier, a tool to enable me to get more bang for my buck. These benefits are made possible by the mantra of “why reinvent the wheel?”. Because Semantic UI is a very flexible and open-source framework, I have the freedom to modify it as I please to fit my needs.

Overall, I’m excited to become more skilled at using Semantic UI so I can be more effective in hacking together my next project.
